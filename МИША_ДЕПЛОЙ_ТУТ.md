# 🚀 МИША, ДЕПЛОЙ ТУТ!

Это простая и понятная шпаргалка, как обновлять твой сайт `af-sport.ru` на сервере после того, как ты внес изменения в код (поправил баги, добавил фичи).

---

## ⚡️ БЫСТРЫЙ ДЕПЛОЙ (Всего 3 команды)

Открой терминал VS Code (или обычный PowerShell на компьютере) в папке проекта `app_mvp` и выполни по очереди:

### Шаг 1. Добавляем все изменения в Git
```powershell
git add -A
```

### Шаг 2. Сохраняем изменения (коммит) и отправляем в GitHub
Замени текст в кавычках на описание того, что ты сделал:
```powershell
git commit -m "fix: поправил баги в профиле и играх"
git push
```

### Шаг 3. Запускаем деплой на сервере
Эта команда подключится к серверу, скачает обновления из GitHub, сама всё пересоберёт и перезапустит сайт:
```powershell
ssh root@91.229.8.235 '/var/www/af-sport/deploy.sh'
```
*У тебя попросит пароль от сервера — введи его. Сборка займет около 1–2 минут.*

---

## 🔍 КАК ПОСМОТРЕТЬ ЛОГИ САЙТА В РЕАЛЬНОМ ВРЕМЕНИ?

Если сайт ведёт себя странно и ты хочешь увидеть, что происходит «под капотом»:

1. Подключись к серверу через SSH:
   ```powershell
   ssh root@91.229.8.235
   ```
2. Посмотри логи приложения:
   ```bash
   pm2 logs af-sport --lines 100
   ```
3. Чтобы выйти из просмотра логов, нажми: `Ctrl + C`.

---

## 🛠 ЧТО ДЕЛАТЬ, ЕСЛИ ЧТО-ТО СЛОМАЛОСЬ?

Если после деплоя сайт упал, ты можешь перезапустить его вручную:

1. Зайди на сервер: `ssh root@91.229.8.235`
2. Перезапусти приложение: `pm2 restart af-sport`
3. Перезапусти веб-сервер Nginx: `systemctl restart nginx`
