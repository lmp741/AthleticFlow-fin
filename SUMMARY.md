# Athletic Flow — простым языком

Что мы делаем, что уже работает, что осталось.

---

## Что за продукт

**Athletic Flow** — сайт для любительского спорта в Москве. Сценарий:
1. Юзер заходит, выбирает вид спорта и район.
2. Видит открытые игры рядом — футбол, баскетбол, волейбол и т.д.
3. Нажимает «Записаться» → попадает в команду на конкретную игру.
4. Игроки скидываются на аренду стадиона (платят свою долю через сайт).
5. После матча — рейтинги, голы, статистика.

Если подходящих игр нет — пользователь сам создаёт игру, выбирает стадион и время, остальные присоединяются.

## Что уже работает (сделали)

### Базовый функционал
- Регистрация / вход через телефон или email.
- Каталог игр с фильтрами (вид спорта, дата, уровень, расстояние).
- Каталог стадионов Москвы (120+ площадок).
- Карта с площадками и дворовыми пятнами из OpenStreetMap.
- Создание своей игры — выбираешь стадион, время, уровень, состав.
- Профиль игрока — никнейм, аватар, уровень, рейтинг.
- Чаты внутри игры + личные сообщения + групповые беседы.
- Френдлист + приглашения друзей на игру.
- Система голов с подтверждением от партнёров (3 подтверждения = подтверждено).
- Уведомления (заявки в друзья, приглашения на игры, ближайшие события).

### Админка
- Дашборд (счётчики: юзеров, забанено, игр, голов на модерации).
- Раздел «Пользователи» — поиск, бан/разбан с причиной, правка username/display_name/level, выдача роли admin.
- Раздел «Игры» — список всех игр, фильтры (будущие / прошедшие / все), отмена, удаление.
- Раздел «Голы» — модерация заявок, force approve / reject.
- Аудит-лог — все действия админов фиксируются (кто, кому, что, когда).

### SEO / GEO под Яндекс
- Главная, /games, /stadiums, /stadiums/[id] — оптимизированы под Яндекс.
- Sitemap.xml с динамическими стадионами.
- llms.txt для нейропоисковиков (Я.Нейро, ChatGPT, Perplexity).
- robots.txt с явным разрешением для GPTBot, ClaudeBot, PerplexityBot, YandexBot и других.
- JSON-LD структурированные данные на страницах стадионов (SportsActivityLocation + FAQPage).
- Meta-теги geo.region, geo.placename, Content-Language ru-RU.

### Безопасность
- RLS политики в supabase — мутации проверяются на бэке, не на клиенте.
- RPC-функции с SECURITY DEFINER — даже если RLS обойти, функция проверит роль сама.
- Защита системы голов от абуза (нельзя заявить за чужого, до конца матча, нельзя одобрить свою заявку).
- Sanitize ввода — regex для username, length limits, escape для поиска.
- JWT auth — не подвержен CSRF.
- Триггер «нельзя удалить последнего admin» — защита от lock-out.

### Юр. часть
- Страница политики конфиденциальности `/privacy` (текст по 152-ФЗ).
- Страница согласия на обработку персональных данных `/personal-data`.
- Ссылки на обе в футере.

4. **Заменить `httpsaf-sport.lovable.app` на свой домен** — 14 мест в коде (см. DEPLOY.md).

5. **Деплой** — на Cloudflare Pages (рекомендую) или Vercel. Бесплатно для MVP.

6. **Платёжный провайдер** — для реальной оплаты участия. Юкасса / Тинькофф эквайринг. Сейчас «оплата» это просто галочка `paid: true` в БД, без денег.

### Рекомендую сделать до промо

7. **Cloudflare Access** для /admin/* — защита админки через email OTP, 10 минут настройки, бесплатно. КРИТИЧНО для безопасности.

8. **Supabase Pro** ($25/мес) — когда пойдут юзеры. Free план не вытянет: лимит 50K MAU, нет бэкапов.

9. **Я.Вебмастер + Google Search Console** — после покупки домена, для индексации.

10. **Яндекс.Метрика** — счётчик посещений, поведенческие данные.

11. **Нагрузочный тест** — `k6 run scripts/loadtest.js` на staging. Чтобы убедиться, что выдержит пиковую нагрузку.

12. **Sentry** или GlitchTip — ловит JS-ошибки у пользователей. Без него ты не узнаешь что у юзера белый экран.

### Со временем

13. **Реальная аренда стадионов** — сейчас модель такая: «организатор сам бронирует, мы только собираем команду». Это OK для старта. Когда наберёшь объёмы — договариваешься с топ-5 стадионами на партнёрство.

14. **PWA / Mobile app** — у тебя в стеке Capacitor, готов к нативной сборке для iOS / Android. Делать когда веб MVP будет стабилен.

15. **Push-уведомления** — Capacitor поддерживает, нужно прикрутить Firebase Cloud Messaging.

16. **Геокодер** — сейчас на Я.Геокодере. На 5К пользователей бесплатных 1000 запросов/сутки не хватит. Перейти на платный тариф или поднять собственный кэш более агрессивно.

## Архитектура простыми словами

**Где код:** в твоём проекте `app_mvp/` (TanStack Start = современный React фреймворк, аналог Next.js).

**Где данные:** в Supabase — это онлайн PostgreSQL + готовый auth + realtime + storage. Один сервис, минимум кода для бэка.

**Где работает:**
- **Сейчас (dev):** на твоём компьютере `localhost:3000`.
- **Скоро (prod):** на Cloudflare Pages — это глобальная сеть серверов, твой сайт будет грузиться быстро во всём мире.

**Кто хранит файлы (аватары и т.д.):** Supabase Storage (когда настроим).

**Кто шлёт смс:** провайдер SMS, который ты подключишь к Supabase Auth (Twilio / Vonage / российские типа SMSc.ru).

**Кто принимает оплату:** платёжный провайдер (Юкасса / Тинькофф) — пока не подключено.
