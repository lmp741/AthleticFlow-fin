## Что добавляем в раздел «Общение»

### 1. Архив диалогов (как в Telegram)
- В списке `/chats` у каждой строки добавляется свайп/кнопка действия: «В архив» / «Из архива».
- Сверху появляется свёрнутая плашка «Архив (N)» — клик раскрывает архивные чаты.
- Архив **личный для каждого пользователя** (как в Telegram): мой архив не виден собеседнику.

### 2. Удаление диалогов
Пункт меню на каждом чате с двумя вариантами:
- **«Удалить у себя»** — скрывает диалог только у меня. Сообщения остаются у собеседника. История восстанавливается, если придёт новое сообщение от него.
- **«Удалить у обоих»** — доступно только в личных DM (не в групповых). Удаляет переписку у меня и у собеседника физически.
  - Для групповых бесед эта опция не показывается; там работают «Покинуть» и (для создателя) «Удалить беседу» — это уже сделано.

### 3. Редактирование и удаление сообщений
Доступно автору сообщения, в личных DM и в групповых беседах:
- **Редактировать** — клик «карандашик» → инпут с текстом → сохранить. Появляется метка «(изм.)» рядом с временем.
- **Удалить** — для своих сообщений в обоих режимах (своё/у всех в DM, своё/у всех в группе для создателя).
- Ограничение: редактировать можно только текстовые сообщения; удалять — любые свои.

### 4. «Избранное» (чат с самим собой)
- Новый раздел в `/chats`, всегда первый в списке, иконка-закладка, нельзя архивировать или удалить.
- Технически — отдельная таблица `saved_messages` (только текст/медиа/файлы/гео, как в обычной DM, но без получателя).
- В неё можно пересылать или просто писать заметки.

### 5. Переименование чатов с людьми
В Telegram-стиле — это **локальный псевдоним**: я переименовываю чат у себя, собеседник этого не видит. Применяется к личным DM (групповые уже умеют переименовываться создателем).
- Сохраняется в новой таблице `chat_aliases (owner_id, peer_id, alias)`.

### 6. Страница пользователя с медиа и контактами
Расширяю существующую `/u/$username`:
- Большой аватар + имя + ник + numeric ID + уровень.
- Блок «Медиа» — сетка из `profile_media` (фото/видео).
- Блок «Контакты» — телефон, **только если** в профиле включён флаг «показывать телефон публично» (новое поле `phone_public`, по умолчанию false).
- Кнопки: «Написать» (открывает DM), «В друзья» (если ещё не друзья), «Оценки» (то, что уже есть).
- В `/profile` добавляю переключатель «Показывать номер телефона на моей странице».

## Технические детали

### Миграции БД

**`conversations`**
- `archived_by uuid[]` — массив id пользователей, которые архивировали беседу.
- `hidden_by uuid[]` — массив id пользователей, у кого диалог скрыт («удалить у себя»). При новом сообщении автоматически очищается из массива для отправителя/всех участников.
- `is_self boolean default false` — отметка для «Избранного» (групповая беседа из одного участника).

**`conversation_messages`**
- `edited_at timestamptz null` — метка редактирования.
- `deleted_at timestamptz null` — мягкое удаление.
- RLS: участник может UPDATE/DELETE только своих сообщений.

**`direct_messages`**
- Аналогично: `edited_at`, `deleted_at`.
- Доп. поля `archived_by_sender bool`, `archived_by_recipient bool`, `hidden_by_sender bool`, `hidden_by_recipient bool` для архива/скрытия DM-диалога с конкретным человеком.
- Новая RLS-политика: «удалить у обоих» = автор сообщения может DELETE; «удалить у себя» — только для DM, реализуется через `hidden_by_*` флаги, а не реальный DELETE.

**Новая таблица `chat_aliases`**
```
owner_id uuid, peer_id uuid, alias text, PRIMARY KEY (owner_id, peer_id)
```
RLS: владелец читает/пишет/удаляет только свои строки.

**`profiles`**
- Новая колонка `phone_public boolean default false`.
- RLS на чтение телефона: телефон возвращается публично только когда `phone_public = true` (через view `public_profiles` или фильтрация на клиенте — выберу clientside-фильтр, потому что менять политики `profiles` рискованно).

### Frontend
- `src/routes/chats.tsx`: секции «Избранное» / «Активные» / «Архив (свёрнут)», dropdown-меню действий на каждой строке (архив, удалить у себя, удалить у обоих, переименовать).
- `src/routes/chats_.$conversationId.tsx`: hover-меню на сообщении (редактировать/удалить), показ «(изм.)».
- Новый файл `src/routes/chats_.dm.$peerId.tsx` уже отсутствует — DM сейчас идут через conversations? Проверю при имплементации; если DM в `direct_messages`, добавлю отдельную страницу или переиспользую существующую.
- `src/routes/u.$username.tsx`: добавляю медиа-сетку, блок контактов, кнопки «Написать» / «В друзья».
- `src/routes/profile.tsx`: переключатель `phone_public`.

### Что НЕ делаю в этой итерации
- Пересылка сообщений в Избранное (можно добавить позже одной кнопкой).
- Закреп сообщений, реакции, ответы — это отдельные большие фичи.
- Поиск по сообщениям.

1. **«Удалить у обоих» для DM** — нужно ли подтверждение в два клика? *(по умолчанию сделаю с подтверждением)*
2. **Редактирование** — ограничить временем (например, 24 часа после отправки) или разрешить всегда? *(по умолчанию — всегда, как в Telegram Premium)*
3. **Псевдонимы DM** — устраивает ли Telegram-вариант (видно только мне), или нужно «настоящее» переименование, видимое обоим? *(по умолчанию — локальный псевдоним)*
4. **Телефон на странице** — показывать всегда, если включён флаг, или только друзьям? *(по умолчанию — всем, кто открыл страницу, при включённом флаге)*

Если все четыре дефолта подходят — просто скажите «делай», и я начну реализацию. Если что-то поменять — скажите, что именно.
